| Field | Details |
|-------|---------|
| **Team Name** | BihariBabu |
| **Domain** | Intelligent Renewable & Smart Energy Systems |

## Overview

India installed **73 GW** of solar capacity by 2023 and targets **292 GW by 2030**. Yet the average rooftop solar panel produces **30–50% surplus energy** on sunny afternoons with no way to sell it at fair market value. EnergyGrid solves this by creating a real-time neighborhood energy marketplace — turning every prosumer household into both a generator and a trader.

> *"India will add 220 GW of solar in 6 years. Every rooftop needs a marketplace. EnergyGrid is that marketplace."*

## Solution — EnergyGrid

A **peer-to-peer energy trading platform** with three core layers working together:

```
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────────┐
│   AI Layer      │    │  Trading Layer   │    │    Trust Layer      │
│─────────────────│    │──────────────────│    │─────────────────────│
│ Physics (pvlib) │    │  Double Auction  │    │ Solidity Contract   │
│ 48h forecast    │    │  Order Book      │    │ onlyOperator guard  │
│ Confidence band │    │  Price-time      │    │ Immutable TX hash   │
│ Savgol Smooth   │    │  priority match  │    │ Carbon credits      │
│                 │    │  ₹2–₹15 bounded  │    │ Sepolia Testnet     │
└─────────────────┘    └──────────────────┘    └─────────────────────┘
```

### What Makes It Different

- **Not net metering** — sellers earn market rate, not utility's throwaway buyback price
- **Not a centralized exchange** — auction mechanism is the price discovery engine
- **Physics-Based Prediction** — Uses `pvlib` for high-accuracy solar irradiance simulation instead of generic statistical models
- **Async blockchain settlement** — trades execute instantly off-chain; Sepolia TX confirms in background
- **Carbon credit auto-tracking** — every trade records `units × 0.4 kg CO₂` offset, verifiable on-chain
- **Community battery pool** — 50 kWh virtual buffer absorbs surplus when no buyer is live

### End-to-End Flow

```
1. Household submits energy reading (produced kWh + consumed kWh)
        ↓
2. Solar service computes surplus via pvlib physics-based simulation
        ↓
3. AI microservice returns 48h forecast + confidence bands + Savgol-smoothed data
        ↓
4. Order book receives surplus → creates sell order / listing
        ↓
5. Marketplace matching: lowest ask meets highest bid or direct buyer fill
        ↓
6. Carbon credit computed: units × 0.4 kg CO₂ → credited to seller
        ↓
7. Blockchain service writes settled trade to Sepolia (recorded on-chain)
        ↓
8. WebSocket broadcasts updated prices + order book to all dashboards
```

## Database Schema

```sql
CREATE TABLE users (
  id              SERIAL PRIMARY KEY,
  name            TEXT,
  email           TEXT UNIQUE,
  password_hash   TEXT,
  wallet_address  TEXT,
  location        TEXT,
  panel_kw        FLOAT DEFAULT 5
);

CREATE TABLE energy_readings (
  id               SERIAL PRIMARY KEY,
  user_id          INT REFERENCES users(id),
  produced_amount  FLOAT,
  consumed_amount  FLOAT,
  recorded_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE listings (
  id              SERIAL PRIMARY KEY,
  user_id         INT REFERENCES users(id),
  amount          FLOAT,
  price_per_unit  FLOAT,
  status          TEXT DEFAULT 'active', -- 'active', 'sold'
  created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE transactions (
  id          SERIAL PRIMARY KEY,
  listing_id  INT REFERENCES listings(id),
  buyer_id    INT REFERENCES users(id),
  seller_id   INT REFERENCES users(id),
  amount_kwh  FLOAT,
  price_inr   FLOAT,
  tx_hash     TEXT,
  created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## Getting Started

### Prerequisites

```bash
Node.js >= 18
Python >= 3.10
PostgreSQL >= 14 (Neon.tech recommended)
Truffle (for blockchain development)
```

### 1. Clone the Repository

```bash
git clone https://github.com/BihariBabu/energygrid.git
cd energygrid
```

### 2. Environment Variables

Create `.env` in `server/`:
```env
PORT=4000
DATABASE_URL=postgresql://user:password@host/dbname?sslmode=require
JWT_SECRET=your-secret
FRONTEND_URL=http://localhost:5173
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret
RPC_URL=https://eth-sepolia.g.alchemy.com/v2/YOUR_KEY
M_PRIVATE_KEY=your_wallet_private_key
DEPLOYED_SMART_CONTRACT=deployed_contract_address
```

### 3. Backend (Node.js)

```bash
cd server
npm install
npm start
# runs on http://localhost:4000
```

### 4. AI Service (Python)

```bash
cd ml
pip install -r requirements.txt
python api.py
# runs on http://localhost:5001
```

### 5. Frontend (React + Vite)

```bash
cd client
npm install
npm run dev
# runs on http://localhost:5173
```

### 6. Smart Contract (Truffle)

```bash
cd server/BlockChain
truffle compile
truffle migrate --network sepolia
```
